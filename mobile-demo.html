体育教师与学生移动端功能规划

老师端以班级管理为主，学生端以打卡与作业为主。右侧手机模型同一页面中用标签标记两种视角。

📱 移动端功能说明

业务流程

1. 课表安排
• 系统已有课表
• 明确上课教师
• 明确班级时间
2. 设备选择项目
• 老师在硬件上操作
• 选择测试项目
• 设备开始准备
3. 学生测试
• 学生依次测试
• 数据自动采集
• 实时同步到云端
4. 手机监控
• 查看实时进度
• 监控学生成绩
• 处理异常情况

数据流向

课表系统
→ 教师信息
→ 班级信息
→ 上课时间
→ 课堂节次
硬件设备
→ 测试项目（老师选择）
→ 测试成绩（自动采集）
→ 测试视频（自动录制）
✕ 不回传设备心跳/微状态
系统处理
→ 成绩等级判定
→ 统计数据计算
→ 异常预警生成
→ 数据推送到手机
